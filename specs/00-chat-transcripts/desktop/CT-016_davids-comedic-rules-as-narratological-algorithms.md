## Larry David's Narratological Algorithms
### *Derived from "Comedy Geometry: Larry David's Architecture of Cascading Consequences"*

---

### 0. Meta-Principles (Axioms)

| Axiom | Formulation |
|-------|-------------|
| LD-A0 | **"No hugging, no learning."** Characters never develop, mature, or learn from mistakes. |
| LD-A1 | Consequences arise from **causal chains**, not ethical judgments. The universe is amoral. |
| LD-A2 | **Comedy geometry** = the mathematical arrangement of story intersections where actions in one plot mechanically trigger disasters in another. |
| LD-A3 | Satisfaction comes from **pattern completion**, not justice. |
| LD-A4 | The collision should feel **inevitable in retrospect, surprising in the moment**. |

---

### 1. Structural Hierarchy

```
EPISODE
  └── STORYLINE (3-5 parallel, apparently independent)
        └── SCENE (cross-cut at increasing pace)
              └── BEAT (improvised dialogue, engineered architecture)
                    └── BRIDGE ELEMENT (planted connector)
```

**Definition Table:**

| Unit | Definition | Constraint |
|------|------------|------------|
| **Storyline** | One character's independent pursuit of a goal | Must be funny *in isolation* AND *in intersection* |
| **Bridge Element** | Object, location, or acquaintance appearing in multiple storylines | Must seem insignificant when planted |
| **Collision Point** | Moment when storylines mechanically intersect | Must be causal, not coincidental at resolution |
| **Retrofit** | Reverse-engineering setups from a discovered collision | Standard practice; non-linear composition |

---

### 2. The Four-Phase Architecture

```
PHASE 1: PARALLEL SETUP (Act I)
├── Establish 3-5 apparently independent storylines
├── Each storyline has internal logic and stakes
├── Plant small details (bridge elements) that seem insignificant
├── Storylines represent different types:
│     WHIM/RAMIFICATION   - impulsive decision → problems
│     PERSONAL_ANTAGONISM - conflict with character
│     GOAL_ORIENTED       - pursuit of objective
└── NO STORYLINE should obviously connect to another

PHASE 2: HIDDEN ARCHITECTURE (Act II)
├── Characters pursue individual goals
├── Introduce bridge elements across storylines:
│     SHARED_LOCATION   (restaurant, event, venue)
│     TRANSFERRED_OBJECT (golf ball, organizer, underwear)
│     MUTUAL_ACQUAINTANCE (person known to multiple characters)
├── Build each storyline's tension independently
├── Apply RETROFIT PLOTTING (see Algorithm 3)
└── Cross-cut between storylines at INCREASING PACE

PHASE 3: MECHANICAL COLLISION (Act III)
├── Bring storylines together through UNINTENDED CONSEQUENCE
├── Collision must be:
│     PHYSICAL/CAUSAL    (not emotional/moral)
│     MECHANICALLY_LINKED (A's action triggers B's consequence)
│     SURPRISING_YET_INEVITABLE
└── BIGGEST LAUGH should occur at moment of collision

PHASE 4: AMORAL RESOLUTION
├── Restore social equilibrium
├── Characters DO NOT learn
├── Consequences proportionate to mechanics, not morality
├── Final moment COMPLETES PATTERN (not delivers judgment)
└── Characters end exactly where they started emotionally/morally
```

### 3. Retrofit Plotting Algorithm

```python
def retrofit_plot(episode):
    """
    Larry David's reverse-engineering method.
    "It's great to know how the scene's going to end.
     Then you sprinkle in stuff beforehand."
    """
    
    # Step 1: Discover or design collision point
    collision = discover_satisfying_collision(storylines)
    
    # Step 2: Determine required bridge elements
    bridges = identify_necessary_connections(collision)
    
    # Step 3: Plant bridges in early scenes
    for bridge in bridges:
        scene = find_organic_placement(bridge, act=1)
        plant_insignificantly(bridge, scene)
    
    # Step 4: Verify causality chain
    for storyline in episode.storylines:
        assert storyline.contributes_to(collision)
        assert not storyline.reveals_connection_prematurely()
    
    return episode
```

**Key Insight**: "We don't have the golf ball goes into the blowhole of the whale... We saw no connection the night before." Connections are often discovered *after* storylines are drafted.

### 4. The Storyline Independence Test

```
FOR each storyline S in episode:
    
    TEST_1: INTERNAL_VALIDITY
    │  Does S have its own logic, stakes, and arc?
    │  IF NO → Storyline is underdeveloped
    
    TEST_2: ISOLATION_COMEDY
    │  Is S funny when viewed in complete isolation?
    │  IF NO → Strengthen independent comedy
    
    TEST_3: INTERSECTION_COMEDY  
    │  Is S funny in how it intersects with others?
    │  IF NO → Redesign collision mechanics
    
    TEST_4: INVISIBILITY
    │  Can audience predict how S connects to others?
    │  IF YES → Hide connections better
```

### 5. Collision Design Rules

```
RULE_1: CAUSAL_NOT_COINCIDENTAL
│  One character's action must MECHANICALLY TRIGGER
│  another character's consequence.
│  
│  VALID:   Kramer hits golf balls → ball lodges in whale →
│           George must extract it
│  INVALID: Characters happen to meet at same location
│           with no causal linkage

RULE_2: PHYSICAL_NOT_EMOTIONAL
│  Connection should be traceable through objects,
│  locations, or physical causation.
│  
│  "The linkage should be physical/causal, not emotional/moral."

RULE_3: CLIMACTIC_PLACEMENT
│  "The hardest thing in comedy is to have the biggest
│   laugh at the end, and it's the most satisfying thing."

RULE_4: INVERSION_OPTION
│  Storylines may converge in SHARED FAILURE.
│  (cf. "The Parking Garage": all plots fail together)
```

### 6. The Amoral Karma Engine

```
TRADITIONAL_SITCOM:
    bad_behavior → moral_punishment
    good_behavior → moral_reward
    
LARRY_DAVID_MODEL:
    action → causal_chain → mechanical_consequence
    
DISTINCTION:
    - George extracts golf ball NOT because he deserves redemption
      BUT because Kramer happened to hit Titleists into the ocean.
    - Elaine loses job NOT because she's a bad person
      BUT because she bought Jujyfruits at the wrong moment.

IMPLEMENTATION:
    consequence = trace_causal_chain(action)
    # NOT: consequence = judge_moral_worth(character)
```

**Berg's Formulation**: "People behave the way they actually behave. They do something bad, and when they get caught they lie about it, and they try to cover it up by doing something else that's worse. And eventually they get discovered, and instead of apologizing, they get f***ed for it."

### 8. Structural Diagnostic Questions

Before finalizing any episode, apply:

| # | Question | Target |
|---|----------|--------|
| 1 | What object, location, or person can connect multiple storylines without characters knowing? | Bridge elements |
| 2 | What action in Storyline A could mechanically affect Storyline B's outcome? | Causal linkage |
| 3 | Working backwards: what collision would be most satisfying? What setups does that require? | Retrofit validation |
| 4 | Is each storyline funny independently, AND funny in how it intersects? | Dual comedy test |
| 5 | Does the resolution feel like cause-and-effect, or like cosmic moral judgment? | Amoral karma check |

### 9. Plotline Type Classification

```
TYPE_1: WHIM_RAMIFICATION
│  Impulsive decision creates escalating problems.
│  Example: Larry discovers Palestinian restaurant.

TYPE_2: PERSONAL_ANTAGONISM  
│  Conflict with recurring or new character.
│  Example: Marty Funkhouser's Orthodox return.

TYPE_3: GOAL_ORIENTED
│  Character pursues specific objective.
│  Example: Richard Lewis needs kidney transplant.

CONSTRAINT: Each episode should include MULTIPLE types
            distributed across storylines.
```

### 10. Episode Templates

**CONVERGENT (Standard)**
```
Setup:    4 independent storylines, 4 characters
Middle:   Cross-cutting, escalating tension per line
Climax:   Single moment where all collide
Example:  "The Marine Biologist"
```

**SHARED FAILURE**
```
Setup:    Characters in shared constrained space
Middle:   Each has independent deadline/need
Climax:   ALL fail together
Example:  "The Parking Garage"
```

**THEMATIC CONVERGENCE**
```
Setup:    Single thematic challenge, multiple responses
Middle:   Each character faces tailored temptation
Climax:   Social faux pas collapses multiple lines
Example:  "The Contest"
```

**HOURGLASS (Inverse Fortunes)**
```
Setup:    Character A declining, Character B ascending
Middle:   Fortunes systematically invert
Climax:   Complete reversal through mechanical causes
Example:  "The Opposite"
```
