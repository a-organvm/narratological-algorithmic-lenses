## 1. Macro: The Omega Vision

The **Omega State** of this project is a computational system capable of ingesting *any* narrative artifact (screenplay, novel, comic script, oral history) and analyzing it through *any* cultural or individual structural framework.

It is not just a linter for scripts; it is a **"Universal Translator for Story Structure."**

### Core Pillars of Omega
1.  **Universal Ingestion**: Frictionless parsing of all major narrative formats (Fountain, FDX, Markdown, PDF).
2.  **Universal Compendium**: A living library of narratological algorithms spanning all human storytelling traditions (Classical, Eastern, Indigenous, Modern, Interactive).
3.  **The Synthetic Mind**: A "Script Doctor" agent swarm that doesn't just critique but *debates*—allowing users to simulate a writers' room with Tarkovsky, Aristotle, and Phoebe Waller-Bridge.
4.  **Zero-Hallucination Diagnostics**: Deterministic structural metrics (Causal Binding, Reorderability) grounded in empirical data, not just LLM vibes.

## 3. Macro: The Synthesis (Back Again)

How these pieces fit together into the final product suite.

### The Ecosystem
1.  **The Library (`packages/core`)**: The brain. Pure Python, type-safe, dependency-free core logic.
2.  **The Tool (`packages/cli`)**: The hands. Power-user interface for batch processing, CI/CD integration, and deep analysis.
3.  **The Service (`packages/api`)**: The voice. HTTP interface for web apps and MCP servers.
4.  **The Workbench (`packages/web`)**: The face. Visual dashboard for exploring the compendium and visualizing story topology.

### The Omega Workflow
1.  **Draft**: User writes in Fountain (VS Code or Highland).
2.  **Lint**: System flags "weak causal binding" in Scene 42 in real-time.
3.  **Consult**: User asks, "What would Billy Wilder do?"
4.  **Simulate**: System runs a "Screwball Comedy" simulation on Act II.
5.  **Refine**: User accepts changes; system updates the beat map instantly.

We are not just building tools; we are codifying the physics of storytelling.
